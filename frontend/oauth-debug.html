OAuth Callback Troubleshooter

Diagnose and fix issues with your Google OAuth callback URL

Problem Description

You're experiencing issues with the OAuth callback URL:

localhost:4000/auth/google/callback?

The page isn't showing anything after selecting Gmail for authentication.

Common Causes & Solutions

1

Google Cloud Console Configuration

Ensure your redirect URI is properly configured in Google Cloud Console:

http://localhost:4000/auth/google/callback
  • Go to Google Cloud Console → APIs & Services → Credentials
  • Select your OAuth 2.0 Client ID
  • Add http://localhost:4000/auth/google/callback to Authorized redirect URIs
  • Save the changes
    • 2

    Environment Variables Check

    Verify your environment variables are set correctly:

    JWT_SECRET=your_jwt_secret_here
    GOOGLE_CLIENT_ID=your_google_client_id
    GOOGLE_CLIENT_SECRET=your_google_client_secret
    FRONTEND_URL=http://localhost:5173

    Your callback code references these environment variables:

    res.redirect(`${process.env.FRONTEND_URL || "http://localhost:5173"}/dashboard?token=${token}`);
    3

    Passport.js Configuration

    Ensure you've properly configured Passport.js with the Google strategy:

    passport.use(new GoogleStrategy({ clientID: process.env.GOOGLE_CLIENT_ID, clientSecret: process.env.GOOGLE_CLIENT_SECRET, callbackURL: "/auth/google/callback" }, (accessToken, refreshToken, profile, done) => { // Your verification logic here }));
    4

    Server Status Check

    Make sure your server is running on port 4000:

    node server.js

    Server Logs Check

    Check your server logs for any error messages. Common issues include:

    // Missing environment variables Error: Secret key not defined // Incorrect redirect URI Error: redirect_uri_mismatch // Database connection issues Error: Failed to save user

    Network Tab Analysis

    Open your browser's developer tools (F12) and check the Network tab:

    1. Look for failed requests (status code 4xx or 5xx)
    2. Check the response from /auth/google/callback
    3. Verify the redirect is happening correctly

    Code Review

    Your callback code looks mostly correct, but here are a few things to double-check:

    router.get( "/google/callback", passport.authenticate("google", { failureRedirect: process.env.FRONTEND_URL || "http://localhost:5173/login", session: false }), (req, res) => { // Generate JWT token const token = jwt.sign({ userId: req.user._id }, process.env.JWT_SECRET, { expiresIn: "7d", }); // Redirect to frontend with token res.redirect(`${process.env.FRONTEND_URL || "http://localhost:5173"}/dashboard?token=${token}`); } );
  • Ensure req.user is properly populated by Passport
  • Verify that process.env.JWT_SECRET is set
  • Confirm your frontend is running on the expected port (5173)
  • Check that the /dashboard route exists in your frontend application

    • Quick Checklist

  • Google Cloud Console redirect URI configured correctly
  • Environment variables set (JWT_SECRET, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, FRONTEND_URL)
  • Server running on port 4000
  • Frontend running on port 5173 (or FRONTEND_URL)
  • Passport.js properly configured with Google strategy
  • JWT token generation working correctly
  • Frontend dashboard route exists
